## [2.7.0] — 2026-03-15

### Added
- **AI Insights Module**: Natural-language anomaly summaries generated by the AI detector, displayed per anomaly group with severity and confidence scoring.
- **Root Cause Analysis Module**: Automated topology traversal and root cause ranking when an incident is created from anomaly alerts.
- **Service Map**: Live dependency graph showing service interconnections, health status, and anomaly propagation paths.
- **SLO/SLA Module**: Configurable Service Level Objectives with error-budget tracking and breach alerting.
- **Correlation Engine**: Cross-signal analysis that links metrics, logs, and traces to anomaly events for context enrichment.
- **Incident Management**: Full lifecycle management with timeline, comments (internal + system), tag assignment, and state machine (open → acknowledged → investigating → resolved → closed).
- **Alert History**: Persistent log of all alerts with search, filter, and CSV export.
- **Alert Rules Engine**: Rule-based alerting with threshold, rate-of-change, and absence conditions.
- **Notification Pipeline**: Slack and email notifications triggered by alert rules. Configurable channels per rule with severity mapping.
- **Deep Link System**: Grafana deep links generated per service/anomaly with time-range context. Panels for CPU, memory, latency, HTTP errors, and saturation.
- **RBAC System**: Role-based access control with 4 roles (SuperAdmin, Admin, Operator, Viewer) enforced at both API and UI levels.
- **Licensing Module**: Hardware-fingerprinted license validation with tier enforcement (Community, Professional, Enterprise).
- **30% MAD Threshold Guard**: AI anomaly detector rejects groups with fewer than 30% data points above the MAD threshold.
- **Notification Cooldown**: Configurable cooldown per alert rule prevents duplicate notifications during sustained anomaly events.
- **Resolve Timeout**: Automatic alert resolution after configurable period of no new anomaly data.

### Changed
- Grafana deep links use direct dashboard URLs instead of API-proxied redirects.
- Alert notification templates include severity badge, metric values, and direct links.
- Frontend error handling: 401 responses on protected pages redirect to login instead of showing broken UI.
- AI Anomalies page: Graceful handling of authentication expiry with user-facing fallback.
- Backend: Standardized JSON error responses for all routers.

### Fixed
- Notification pipeline: Eliminated duplicate emails when multiple anomaly groups resolve simultaneously within the same cooldown window.
- Notification pipeline: Fixed race condition where incident creation and alert resolution could fire conflicting notifications.
- Alert system: Corrected cooldown timer reset logic that allowed premature re-notification on long-running incidents.
- AI Anomalies page: Fixed 401 error when token expires mid-session.
- Grafana deep links: Fixed incorrect time range calculation for panel URLs.

### Security
- JWT token validation enforced on all API routes (except `/health` and `/auth/login`).
- CORS policy restricted to known frontend origin.
- Rate limiting applied to authentication endpoints.

## [2.5.0] — 2024-11-15

### Added
- **Public Overview Documentation**: First public-facing product documentation.
- **Feature Matrix**: Published capability comparison across license tiers.
- **Branding System**: Rhinometric brand identity applied to frontend with logo, colors, typography.
- **AI Anomaly Detection v1**: Initial deployment of IsolationForest-based detector as containerized Python service.
- **VictoriaMetrics Integration**: Long-term metric storage backend replacing Prometheus retention.
- **Jaeger Tracing**: Distributed tracing collection via OpenTelemetry Collector.
- **Loki Log Aggregation**: Centralized log collection with Promtail agents.

### Changed
- Frontend migrated from Create React App to Vite.
- Authentication switched from session-based to JWT.
- Prometheus reconfigured as short-term buffer (15-day retention) with VictoriaMetrics as long-term store.

## [2.1.0] — 2024-09-01

### Added
- Core service monitoring with Prometheus metric collection.
- Basic alerting via Alertmanager with email notification.
- Grafana dashboards for infrastructure and service metrics.
- Initial frontend with React 18 + TypeScript.
- PostgreSQL database for platform state.
- Redis caching layer.
- Docker Compose deployment with 15 services.
- Nginx reverse proxy with SSL termination.

### Known Issues
- No database migration system.
- Limited to single-node deployment.
- No automated installer.
