📋 RESUMEN EJECUTIVO

Rhinometric es una plataforma integral de observabilidad empresarial que unifica métricas, logs y trazas distribuidas en una solución lista para usar. Construida sobre tecnologías open source líderes de la industria (Grafana, Prometheus, Loki, Tempo), Rhinometric ofrece una experiencia de instalación simplificada con configuración preestablecida y dashboards listos para producción.

Propuesta de Valor

  • 🚀 Instalación en 1-click - Desde cero a observabilidad completa en menos de 5 minutos
  • 📊 Dashboards precargados - 7 dashboards profesionales listos para usar (100% funcionales)
  • 🔒 Seguridad integrada - Sistema de licenciamiento con Time-Bomb (30 días), autenticación segura
  • 💰 Costo predecible - Sin sorpresas en facturación, recursos controlados
  • 🛠️ Soporte técnico - Asistencia profesional incluida durante el trial (24-48h)
  • 🔄 100% funcional - Logs, Traces y Métricas validados en producción

🏗️ ARQUITECTURA DE LA PLATAFORMA

Rhinometric está construido como una arquitectura de microservicios organizada en 7 capas (tiers) con 15 contenedores especializados:

TIER 1: Licenciamiento (1 contenedor)

Contenedor Imagen Puerto Recursos Descripción
license-server Custom Flask 5000 0.3 CPU / 256MB Servidor de licencias: Genera y valida licencias trial/comerciales con firma SHA256. API REST para gestión, integración PostgreSQL.

TIER 2: Base de Datos y Caché (2 contenedores)

Contenedor Imagen Puerto Recursos Descripción
postgres postgres:15 5432 1.0 CPU / 2GB Base de datos relacional: Configuraciones, usuarios, licencias, dashboards. Retención trial: 7 días.
redis redis:alpine 6379 0.2 CPU / 256MB Caché en memoria: Sesiones, queries, rate limiting. Persistencia AOF, evicción LRU.

TIER 3: Observabilidad Core (5 contenedores)

Contenedor Imagen Puerto Recursos Descripción
prometheus prom/prometheus 9090 1.0 CPU / 2GB Motor de métricas: Recolecta métricas cada 15s. Retención 7 días. 16 reglas de alertas activas.
loki grafana/loki 3100 0.5 CPU / 1GB Agregador de logs: Indexa logs de TODOS los contenedores. Compatible LogQL. Extrae container_id automático.
tempo grafana/tempo 3200, 4317 0.5 CPU / 512MB Distributed tracing: Captura trazas OTLP/Jaeger/Zipkin. Generando ~15 spans/seg continuo.
telemetrygen otel/telemetrygen - 0.1 CPU / 64MB Generador de trazas: Ejecución infinita, 5 spans/seg, 3 workers. Envía trazas sintéticas continuas.
grafana grafana/grafana 3000 0.8 CPU / 1GB Dashboard principal: 7 dashboards precargados, autenticación segura (admin_secure_2024).

TIER 4: Alerting (1 contenedor)

Contenedor Imagen Puerto Recursos Descripción
alertmanager prom/alertmanager 9093 0.3 CPU / 512MB Gestor de alertas: Recibe alertas de Prometheus, agrupa, envía notificaciones (email, Slack).

TIER 5: Exporters de Métricas (4 contenedores)

Contenedor Imagen Puerto Recursos Descripción
node-exporter prom/node-exporter 9100 0.2 CPU / 128MB Métricas del host: CPU, RAM, disco, red, procesos. 1000+ métricas del SO.
cadvisor gcr.io/cadvisor 8080 0.3 CPU / 256MB Métricas de contenedores: Uso de recursos por contenedor (CPU, RAM, I/O, red).
blackbox-exporter prom/blackbox 9115 0.1 CPU / 64MB Probes HTTP/TCP: Monitorea disponibilidad de URLs, latencia, certificados SSL.
postgres-exporter prometheus/postgres 9187 0.1 CPU / 64MB Métricas PostgreSQL: Conexiones, queries lentas, locks, cache hit ratio.

TIER 6: Dashboard de Licencias (1 contenedor)

Contenedor Imagen Puerto Recursos Descripción
license-dashboard Custom Python 8080 0.3 CPU / 256MB Panel de licencias: Visualiza licencias activas, expiración, estadísticas. Solo lectura en trial.

TIER 7: Proxy y Colector de Logs (2 contenedores)

Contenedor Imagen Puerto Recursos Descripción
nginx nginx:alpine 80, 443 0.3 CPU / 256MB Reverse proxy: Punto de entrada único. Enrutamiento, SSL/TLS, compresión, rate limiting.
promtail grafana/promtail 9080 0.2 CPU / 128MB Agente de logs: static_configs (compatible WSL2), extrae container_id automático. Envía a Loki.

🎯 ALCANCE DE LA PLATAFORMA

Casos de Uso Principales

1. Monitoreo de Infraestructura

  • Supervisión en tiempo real de servidores (CPU, RAM, disco, red)
  • Alertas automáticas ante degradación de rendimiento
  • Visualización de tendencias y patrones de uso

Ideal para: Equipos DevOps, SREs, Administradores de sistemas

2. Observabilidad de Aplicaciones

  • Métricas de negocio (usuarios activos, transacciones, errores)
  • Trazado distribuido para identificar cuellos de botella
  • Debugging de aplicaciones sin acceso a servidores

Ideal para: Desarrolladores, Product Managers, QA

3. Monitoreo de Microservicios

  • Mapa de servicios con dependencias automáticas
  • Latencia P50/P95/P99 por endpoint
  • Service Level Indicators (SLI) y Objectives (SLO)

Ideal para: Arquitectos de software, Equipos de microservicios

Tipos de Proyectos Soportados

✅ E-commerce y Retail
Monitoreo de transacciones, alertas de caídas de conversión, análisis de tráfico en campañas
✅ Fintech y Banca
Trazabilidad completa de transacciones, compliance con auditorías, detección de anomalías
✅ SaaS y Plataformas Web
Métricas de multi-tenancy, facturación basada en uso, SLA monitoring
✅ Aplicaciones Móviles (Backend)
Monitoreo de APIs REST/GraphQL, latencia por región, errores por versión de app
✅ IoT y Dispositivos
Ingesta de métricas de miles de dispositivos, alertas por desconexión

Instalación Simplificada

Windows

  1. Descargar RhinometricInstaller.exe
  2. Ejecutar instalador (requiere permisos de administrador)
  3. El wizard verifica Docker, instala si es necesario
  4. Instalación automática (5-8 minutos)
  5. Grafana se abre automáticamente al finalizar

macOS

  1. Descargar Rhinometric.dmg
  2. Arrastrar a carpeta Aplicaciones
  3. Ejecutar Rhinometric.app
  4. Seguir wizard de instalación

Linux

# Ubuntu/Debian
wget https://rhinometric.com/downloads/rhinometric-trial.deb
sudo dpkg -i rhinometric-trial.deb
rhinometric-install

# RHEL/CentOS
wget https://rhinometric.com/downloads/rhinometric-trial.rpm
sudo rpm -i rhinometric-trial.rpm
rhinometric-install
