Comenzar Instalación

✅ Requisitos del Sistema

Antes de instalar Rhinometric, verifica que tu sistema cumpla con los siguientes requisitos:

💾

Memoria RAM

Mínimo: 8 GB | Recomendado: 16 GB

⚙️

Procesador

Mínimo: 4 cores | Recomendado: 8 cores

💿

Disco Duro

Mínimo: 50 GB SSD | Recomendado: 100 GB SSD

🐳

Docker Desktop

Versión 20.10 o superior (el instalador puede instalarlo automáticamente)

💡 Nota: El instalador de Rhinometric verifica automáticamente estos requisitos y te guiará en caso de que falte algo.

📦 Instalación por Sistema Operativo

Selecciona tu sistema operativo para ver las instrucciones específicas:

Instalación en Windows 10/11

Descargar el Paquete Trial

Descarga rhinometric-trial-v1.0.0-production.zip desde:

  • 📍 Tu proveedor (email, USB, Google Drive, etc.)
  • 🔗 O desde: rhinometric.com/downloads/trial

Tamaño: ~41 KB (muy ligero)

Extraer el Archivo ZIP

Ubicación recomendada: C:\rhinometric-trial

  1. Clic derecho en rhinometric-trial-v1.0.0-production.zip
  2. Seleccionar "Extraer todo..."
  3. Cambiar ruta a C:\rhinometric-trial
  4. Clic en "Extraer"

Deberías tener:

C:\rhinometric-trial\rhinometric-trial-v1.0.0-production\

Abrir PowerShell en la Carpeta

  1. Abre la carpeta C:\rhinometric-trial\rhinometric-trial-v1.0.0-production
  2. En el explorador de archivos, haz clic en la barra de dirección
  3. Escribe powershell y presiona Enter

Alternativamente: Shift + Clic derecho en espacio vacío → "Abrir ventana de PowerShell aquí"

Instalación en macOS (Intel y Apple Silicon M1/M2/M3)

Descargar el Paquete Trial

Descarga rhinometric-trial-v1.0.0-production.zip desde tu proveedor o:

rhinometric.com/downloads/trial

Tamaño: ~41 KB

Extraer el Archivo

Ubicación recomendada: ~/rhinometric-trial

  1. Doble clic en el archivo ZIP (se extrae automáticamente)
  2. O desde Terminal:
# Crear carpeta y extraer
mkdir -p ~/rhinometric-trial
unzip rhinometric-trial-v1.0.0-production.zip -d ~/rhinometric-trial
cd ~/rhinometric-trial/rhinometric-trial-v1.0.0-production

Verificar Docker Desktop

Asegúrate de que Docker Desktop esté corriendo:

  1. Busca el ícono de Docker en la barra de menú superior
  2. Debe decir "Docker Desktop is running"
  3. Si no está instalado, descárgalo de: docker.com/products/docker-desktop
⚠️ Apple Silicon (M1/M2/M3): Asegúrate de instalar la versión "Apple Chip" de Docker Desktop.

Iniciar Rhinometric

Abre Terminal y ejecuta:

cd ~/rhinometric-trial/rhinometric-trial-v1.0.0-production
docker compose up -d

Espera 2-5 minutos mientras se descargan e inician los 16 contenedores.

Verificar estado:

docker ps | grep rhinometric

Deberías ver 16 líneas (todos los contenedores).

Abrir Grafana

  1. Abre Safari, Chrome o tu navegador favorito
  2. Ve a: http://localhost:3000
  3. Login:

Usuario: admin

Contraseña: admin_secure_2024

🎉 ¡Listo! Explora los 7 dashboards.

Detener:

docker compose down

{job="docker_logs"} |= "error"

{job="docker_logs", container_id=~".*grafana.*"} |= "error"

🔌 Conectar tus Aplicaciones a Rhinometric

Rhinometric puede recolectar métricas, logs y trazas de tus aplicaciones. Aquí te mostramos cómo integrarlas:

Python (Flask / FastAPI)

1. Instalar librerías

pip install prometheus-client opentelemetry-api opentelemetry-sdk opentelemetry-instrumentation-flask

2. Configurar métricas (Prometheus)

from prometheus_client import Counter, Histogram, generate_latest
from flask import Flask, Response
import time

app = Flask(__name__)

# Crear métricas
REQUEST_COUNT = Counter('app_requests_total', 'Total de requests', ['method', 'endpoint'])
REQUEST_LATENCY = Histogram('app_request_latency_seconds', 'Latencia de requests')

@app.before_request
def before_request():
    request.start_time = time.time()

@app.after_request
def after_request(response):
    latency = time.time() - request.start_time
    REQUEST_COUNT.labels(method=request.method, endpoint=request.endpoint).inc()
    REQUEST_LATENCY.observe(latency)
    return response

# Endpoint para Prometheus
@app.route('/metrics')
def metrics():
    return Response(generate_latest(), mimetype='text/plain')

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)

3. Configurar en Prometheus

Edita config/prometheus-saas.yml y agrega:

scrape_configs:
  - job_name: 'mi-app-python'
    static_configs:
      - targets: ['host.docker.internal:5000']  # Windows/Mac
      # - targets: ['172.17.0.1:5000']  # Linux

4. Reiniciar Prometheus

docker restart rhinometric-prometheus

Ahora puedes ver métricas de tu app en Grafana con queries como:

rate(app_requests_total[5m])

Node.js (Express)

1. Instalar librerías

npm install prom-client express-prom-bundle

2. Configurar métricas

const express = require('express');
const promBundle = require('express-prom-bundle');

const app = express();

// Middleware de métricas
const metricsMiddleware = promBundle({
    includeMethod: true,
    includePath: true,
    includeStatusCode: true,
    promClient: {
        collectDefaultMetrics: {
            timeout: 5000
        }
    }
});

app.use(metricsMiddleware);

// Tu app aquí
app.get('/', (req, res) => {
    res.send('Hello World!');
});

app.listen(3001, () => {
    console.log('App running on port 3001');
    console.log('Metrics available at http://localhost:3001/metrics');
});

3. Agregar a Prometheus

scrape_configs:
  - job_name: 'mi-app-nodejs'
    static_configs:
      - targets: ['host.docker.internal:3001']

Java (Spring Boot)

1. Agregar dependencia (pom.xml)

<dependency>
    <groupId>io.micrometer</groupId>
    <artifactId>micrometer-registry-prometheus</artifactId>
</dependency>

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>

2. Configurar application.properties

management.endpoints.web.exposure.include=health,prometheus
management.metrics.export.prometheus.enabled=true

3. Agregar a Prometheus

scrape_configs:
  - job_name: 'mi-app-java'
    metrics_path: '/actuator/prometheus'
    static_configs:
      - targets: ['host.docker.internal:8080']

Go (Gin)

1. Instalar librerías

go get github.com/prometheus/client_golang/prometheus
go get github.com/prometheus/client_golang/prometheus/promhttp

2. Código

package main

import (
    "github.com/gin-gonic/gin"
    "github.com/prometheus/client_golang/prometheus"
    "github.com/prometheus/client_golang/prometheus/promhttp"
)

func prometheusHandler() gin.HandlerFunc {
    h := promhttp.Handler()
    return func(c *gin.Context) {
        h.ServeHTTP(c.Writer, c.Request)
    }
}

func main() {
    r := gin.Default()
    
    // Endpoint de métricas
    r.GET("/metrics", prometheusHandler())
    
    // Tu app aquí
    r.GET("/", func(c *gin.Context) {
        c.JSON(200, gin.H{"message": "Hello World"})
    })
    
    r.Run(":8080")
}
💡 Tip: Para enviar logs a Loki, configura tu app para que escriba logs en formato JSON y usa Promtail para recolectarlos. Para trazas distribuidas, usa librerías de OpenTelemetry para tu lenguaje.

🔧 Solución de Problemas Comunes

❌ "Docker no está corriendo"

Problema: El instalador o script no puede conectarse a Docker

Solución:

  • Windows/Mac: Abre Docker Desktop desde el menú Inicio o Aplicaciones. Espera a que el ícono en la barra de tareas/menú superior diga "Docker is running"
  • Linux: sudo systemctl start docker

❌ "No se puede descargar imágenes Docker"

Problema: Error al descargar grafana/grafana, prom/prometheus, etc.

Causas posibles:

  • Sin conexión a internet
  • Firewall bloqueando Docker
  • Proxy corporativo

Solución:

# Verificar conectividad
docker pull alpine:latest

# Si falla, configurar proxy en Docker Desktop:
# Settings → Resources → Proxies → Manual proxy configuration

❌ "Puerto 3000 ya está en uso"

Problema: Grafana no puede iniciar porque otro servicio usa el puerto 3000

Solución:

  1. Identifica qué usa el puerto: 
    # Windows
netstat -ano | findstr :3000

# Mac/Linux
lsof -i :3000
  2. Detén ese servicio o cambia el puerto de Grafana en docker-compose-trial.yml: 
    grafana:
  ports:
    - "3001:3000"  # Cambiado a 3001
  3. Reinicia: docker compose -f docker-compose-trial.yml up -d

❌ "RAM insuficiente"

Problema: Contenedores se reinician constantemente o el sistema está lento

Solución:

  • Windows/Mac: Docker Desktop → Settings → Resources → Memory → Aumentar a 8 GB
  • Linux: Verificar RAM disponible: free -h
  • Cerrar aplicaciones que consumen mucha RAM

❌ "No puedo acceder a Grafana"

Problema: http://localhost:3000 no carga

Diagnóstico:

# Verificar que Grafana está corriendo
docker ps | grep grafana

# Ver logs de Grafana
docker logs rhinometric-grafana

# Reiniciar Grafana
docker restart rhinometric-grafana

Si ves errores en los logs, revisa que PostgreSQL esté corriendo:

docker ps | grep postgres

❌ "Olvidé mi contraseña de Grafana"

Solución:

  1. La contraseña por defecto es admin_secure_2024 (definida en archivo .env)
  2. Si cambiaste la contraseña y la olvidaste, puedes resetearla: 
    # Entrar al contenedor de Grafana
docker exec -it rhinometric-grafana /bin/bash

# Resetear contraseña del admin
grafana-cli admin reset-admin-password nueva_contraseña_aqui

# Salir del contenedor
exit

# Reiniciar Grafana
docker restart rhinometric-grafana

❌ "No veo métricas de mi aplicación"

Problema: Conectaste tu app pero no aparecen métricas en Grafana

Diagnóstico:

  1. Verifica que tu app expone métricas: curl http://localhost:PORT/metrics
  2. Verifica que Prometheus puede alcanzar tu app: 
    # Ver targets en Prometheus
# Ve a http://localhost:9090/targets
# Tu app debe aparecer con estado "UP"
  3. Si está "DOWN", verifica la URL en config/prometheus-saas.yml
  4. Reinicia Prometheus: docker restart rhinometric-prometheus
